学习Swift开发iOS应用教程第四篇「译」

您的第一个应用程序

Posted by Jian on March 6, 2016

很高兴，你来到Swift教程系列编写iOS应用最终篇！

在系列的第一部分，你学会了Swift编程基础知识。学习了变量，if/else语句，循环，可选类型等等。

在系列的第二部分，你将所学到的技能运用到一个简单的猜数字的游戏中.

在系列的第三部分，你创建了一个记录用户名和年龄的命令行应用。

在系列的第四部分，是时候创建您的第一个 iPhone 应用！

在这一教程中，你会学着创建一个应用，应用名叫 点击我吧(Tap Me) ！该应用的玩法是在30秒的时间内，尽你最快的速度点击应用的按钮，最终应用将会告诉你30秒内点击的次数（其实就是类似微信里面第三方的一些加油活动）。

开始（多图预警）

打开Xcode,点击 Get started with a playground :

然后选择 iOS > Application > Single View Application 点击 Next 。

在下面的屏幕中，根据指示填写表格：

  • Product Name : Tap Me
  • Organization Name : 选填
  • Organization Identifier: 输入 com.yourname, 例如 com.rybristow
  • Language : Swift
  • Devie : iPhone

故事板（Storyboards）

您将会发现这个项目创建成功之后的目录结构跟之前的命令行项目不同。第一个不同点就是下面选中的这个文件 Main.storyboard :

当你开打这个文件，将会印入眼帘的是一个模拟 iPhone 手机运行时的窗口的图形用户界面(GUI)。

注意: 你也许会问为什么 iPhone 的屏幕看起来像个方形。 在 iOS 开发中，你的目标是设计你自己的应用因此它们应该适配各种尺寸的屏幕 - 这叫做 自应式布局（Adaptive Layout） 。基本上，界面元素的布局排版根据 “离顶部距离多远” 或 “居中” 的原则；这种方式布局能够在多种尺寸的屏幕适用，从 iPhone 5 到 iPhone 6 Plus 等等

现在，该应用的界面是空白一片并且非常无趣。做一些事情改变下它，首先确保你选中并打开了右边的工具菜单(Utilities Menu)。

接着，选中属性检查器(Attributes Inspector)。

这样你就可以看到控制器(View Controller)的界面属性以及修改它们。确保你第一个点击的视图(View)能够看到它的属性。

修改视图的背景色为绿色。

现在，在顶部的菜单选择 iPhone 6 模拟器 并运行它，你就能看到运行的界面是什么样子。编译和运行你的程序会花一点时间，请耐心等待。当它运行完成的时候，你将会看到一个窗口弹出，该窗口模拟着 iPhone 手机界面并运行着您的应用程序。

往屏幕增加标签(Labels)

你的程序看上去运行的很好，但是界面太不美观因为它没有任何可以跟用户交互的内容。为界面添加对象，你需要在工具菜单的对象库(Object Library)里找到名叫标签(Label)的对象，标签的功能是显示文本，可以通过程序控制文本内容。滚动对象库直到你找到它为止。

拖动文本标签到屏幕的顶部并水平居中。

你可以使用属性检查器来修改文本标签的标题。修改标签的标题为 “Time: 30”。

此刻你的标签内容在屏幕显示应该是不完全的。为了解决这个问题，点击标签选择 Editor\Size to Fit (Command =快捷键)。

现在，运行程序您应该看到的是：

根据你的故事板布局，标签应该显示在中间才对呀？问题是并不是每一个 iOS 社保拥有着相同的面积，所以有时候不能那么顺心。你需要做的是为标签增加一条约束(constraint)让它一直保持在中间。

要实现这一点，首先点击标签。然后，选择屏幕底部的对齐按钮(Align button)并且勾选水平居中的约束，最后点击添加一条约束(Add 1 Constraint)按钮。

该约束能够让标签固定在水平位置。但是你还需要定义它在垂直方向的位置。要实现这一点，点击屏幕底部的方块按钮(Pin button)并且点击红色的条子让它钉在屏幕的上方，基于它当前跟顶部的距离点击添加一条约束(Add 1 Constraint)按钮：

最新添加的标签约束会延展出一条橘色的参考线。这是因为文本标签的位置跟约束的指定位置不同。

如果这是个污点，点击右下角的三角图标，在 All Views in View Controller 栏目选中 Update Frames 选项。

现在，创建另一个标签在屏幕的底部用来记录点击次数。设置字体大小为 40 ，行数为 2 以及 对齐方式 居中。

在尺寸检查器中，设置 高度 为 130 点。

你最好为这个标签设置一些约束。首先你需要让它居中。选中分数标签，点击对齐按钮，选中 Horizontal Center in Container 并且点击 Add 1 Constraint 。

接着点击方块按钮。固定底部距离并且选中 height 约束。设置高度为 130 。最后，点击 Add 2 Constraints 。

如果你看见了许多橘色参考线，点击右下角的 三角图标 并且在 All Views in View Controller 栏目里选中 Update Frames 。

编译并运行，现在你所有的标签都居中在屏幕上：

添加一个按钮

添加最后一个对象到屏幕，使用对象库添加一个按钮到屏幕并且固定在屏幕的中间。

别忘了为按钮添加约束，这次需要水平和垂直居中两条约束。

现在，使用 属性检查器 来改变按钮的标题为 Tap Me!并且设置背景成白色。为了让按钮显示更多的白色背景，你需要拉伸它并且设置其他约束。这次，你需要点击方块按钮并且设置它的宽为 167 和高为 113 。

你可以尝试疯狂的点击按钮，但是你发现什么事情都没有发生。接下来你会让它有所反应。

连接视图和代码

现在故事板里的视图已经布局好了，你需要让它们跟代码关联起来。

每一个 iOS 应用”屏幕”被一个叫 View Controller（视图控制器） 的东西控制着。Xcode在模板中已经创建了一个 - ViewController.swift 。

你需要在你的游戏中为这个类设置两个标签属性，这样你才能通过程序更新它们的内容。要实现这点，在 ViewController.swift 的视图控制器类中添加以下两个属性：

@IBOutlet var scoreLabel: UILabel!
@IBOutlet var timerLabel: UILabel!

这两行代码创建了两个 UILabel 属性。此刻，你将会连接这两个属性所对应在故事板中的标签视图。通过为属性设置 @IBOutlet 关键字，故事板编辑器就能够识别它们。

注意：你也许察觉到标签的末尾有感叹号。这是标记标签为“强制解包可选类型”。这会使在你访问它们之前，编译器自动解包了可选类型，因此你可以将变量视作-不是可选类型变量，意思是普通变量(哪怕它们是可选变量)。

你可以这样做因为你已经确定你将会将它们设置成故事板里的视图，因此它们不可能为 nil 。

类似地，你希望当按钮点击的时候调用一个函数。因此添加以下代码到 ViewController 类中：

@IBAction func buttonPressed()  {
  NSLog("Button Pressed")
}

跟 @IBOutlet 一样, 添加 @IBAction 关键字能够让故事板识别到该函数，因此你能够将该函数连接按钮的点击事件。

尝试连接这些属性和函数！打开 Main.storyboard ，找到文档大纲，你需要做的是通过按住 CTRL 键并且拖动左边的 View Controller 到在视图顶部的标签。

一个黑色盒子将会弹出输出列表。选中 timerLabel。这是你之前在 ViewController.swift 创建的一个 UILabel 对象。

为 score label 进行同样的操作。但是连接的对象是 scoreLabel。

为按钮添加 buttonPressed() 点击事件，你需要采取反向操作。按住 CTRL 键点击故事板中的按钮拖动连接到左边的文档大纲 View Controller。

然后，选择

现在，尝试运行游戏并且点击几次按钮。因为 buttonPressed() 函数里的 NSLog 语句，你每次点击按钮的时候屏幕底部的控制台将会输出一些日志。

巧妙地控制标签

按钮真正的目的并不是打印日志在控制台，而是每当点击时候改变分数里面的值。首先，你应该学着如何控制标签里面的内容。

好消息 - 这很简单！替换 buttonPressed() 函数里面的代码：

scoreLabel.text = "Pressed"

现在，当你运行你的游戏并且点击按钮，屏幕底部的文本将会改变成 “Pressed”。

为了使用这个技能来让你的分数标签显示正确的分数，你还需要做点东西。

分数记录

为了记录分数，首先你需要创建一个变量。在类的顶部左括号内两个 @IBOutlet 定义下面，创建一个属性 count 来记录分数。=

var count = 0

下一步是在 buttonPressed() 函数顶部进行分数累加：

count++

这一行的缩写的原意是：

count = count + 1

现在，修改代码 _ scoreLabel_ 文本：

scoreLabel.text = "Score \n\(count)"

运行你的游戏再次测试。当你点击按钮的时候，屏幕的底部标签将会记录你的分数。你的程序已经越来越接近一个可玩的游戏了！

倒计时

为了这游戏能有走时准确的时钟，你需要创建两个新属性。一个是 int 类型来记录秒数和一个 NSTimer 来实现倒计时。

在 count 的定义下方加入以下两个变量：

var seconds = 0
var timer = NSTimer()

为了使用这些变量，你需要创建两个函数。第一个是 setupGame()。在类的底部花括内加入以下代码：

func setupGame()  {
  seconds = 30
  count = 0
 
  timerLabel.text = "Time: \(seconds)"
  scoreLabel.text = "Score: \(count)"
}

这块代码是在游戏进行中还原所有变量到原始值。如果你没有加入这些代码，秒数会继续递减至0并且分数将会持续一直递增下去。

下一步还需要为 setupGame() 函数启动一个定时器。为了实现这点，你需要使用 scheduledTimerWithTimeInterval() 函数来获得你想要的定时器。在 setupGame() 加入以下代码。

timer = NSTimer.scheduledTimerWithTimeInterval(1.0, target: self, selector: Selector("subtractTime"), userInfo: nil, repeats: true)

现在，确保你在 viewDidLoad() 函数里调用了 setupGame() 函数：

setupGame()

下一个方法是 subtractTime()。你会发现当你使用 setupGame 里的定时器的时候你需要在选择器中使用这个函数。这就是所涉及道德。这个函数用来递减秒数，更新 timeLabel，当计时器为0时触发一个警报器。

你已经知道事情的顺序：递减 seconds，更新 timerLabel，并且当秒数为 0 的时候创建一个 if 语句。

func subtractTime() {
  seconds--
  timerLabel.text = "Time: \(seconds)"
 
  if(seconds == 0)  {
 
  }
}

在 if 条件语句中第一件要做的事情是停止已经计时为0的计时器。

timer.invalidate()

编译并且运行你的游戏，现在时钟已经会倒计时了!

红色警报!

下一步，你将会学习如何使用警报。当你完成警报的编程时候，你应该看到的界面如下：

是时候干正事了。第一件事创建一个 UIAlertController。在 if 判断内的 invalidate() 语句后面添加：

let alert = UIAlertController(title: "Time is up!",
 message: "You scored \(count) points",
 preferredStyle: UIAlertControllerStyle.Alert)

这一行定义了警报的内容。指定了标题，消息，还有显示的风格。

接着，你需要调整按钮的内容和事件。为了实现这一点，你需要使用 addAction()。在 alert 的定义下面编写如下代码。

alert.addAction(UIAlertAction(title: "Play Again", style: UIAlertActionStyle.Default, handler: {
  action in self.setupGame()
}))

如果你仔细阅读这段代码，你将会发现 title 决定了按钮的显示文字，并且按钮点击会触发 setupGame() 函数，警报框会消失。

最后一条代码!这一句是告诉应用跳出一个 UIAlertController。使用 presentViewController() 函数。在 alert.addAction(…) 之后加入以下代码：

presentViewController(alert, animated: true, completion:nil)

现在运行您的游戏。你能够打败我吗？
