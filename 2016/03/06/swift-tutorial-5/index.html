学习Swift开发iOS应用教程第五篇「译」

美化界面

Posted by Jian on March 6, 2016

很高兴，你来到Swift教程系列编写iOS应用最终篇！

在系列的第一部分，你学会了Swift编程基础知识。学习了变量，if/else语句，循环，可选类型等等。

在系列的第二部分，你将所学到的技能运用到一个简单的猜数字的游戏中.

在系列的第三部分，你创建了一个记录用户名和年龄的命令行应用。

在系列的第四部分，你创建了第一个 iPhone 应用！

在教程的最后一篇，使游戏看起来有点更具有视觉吸引力的怎么样?。

这部分将会学习到如何设置屏幕上的元素背景。而且，你还会学到如何实现播放背景音乐和音响效果。让我们开始吧！

开始

开始前需要打开上一个项目的工程，如果你没有该工程点击下载备份工程。

我建议在开始前先备份一份项目。这样你就可以有一份原始版本进行对比。不仅能比较两个完整版本的内容，而且如果你搞杂了界面，还可以用回原始版本。

然后，你也许想要下载Tap Me资源，该文件包含着该项目需要的一些图片和音乐素材。在你下载完后，你应该打开文件并且将所有的文件放到你的工程目录里。将这些资源拖放到文档大纲的辅助文件夹(Supporting Files)里面。

确保勾选了 Copy items if needed 选项，如果你 移动/删除 了下载的资源文件，这样您的项目在其他的电脑依然也能够使用。

设置按钮的背景图

比起给按钮设置背景色，在图片编辑器中设置了一张图片要更好看的多。要设置背景图片，你需要设置按钮对应的状态。确定按钮的状态在属性检查器中为 Default 。这是普通状态下的按钮。

然后，找到名为 Background 的盒子，设置为 button_tap_deselected.png，

在你上一个版本的应用中，你已经设置过按钮背景成白色。修改背景色为透明 Default ，这样在图片的后面就不会有白色的方块。


糟糕！标题设置成 “Tap Me!” 导致图片看起来效果不好。因为图片里面也有文字，因此你需要将按钮的标题设置为空。


好多了！现在，按钮应该是清秀的并且比之前的一个版本好看很多。不再是一个单调的白色文本，是一个有丰富色彩和好看的文字的3维按钮。

下一步你需要为按钮的高亮状态设置另一张背景图。当用户点击了按钮时候的状态称为高亮状态。高亮状态的背景设置成 button_tap_selected.png 。

你应该察觉到了按钮看起来有些压扁的赶脚。这是因为硬编码的宽高约束比实际上的图片要小一些导致的。

幸运的是，有一个简单的方法解决这个问题。所有的视图都有一个 intrinsic content size(本质内容大小) 的东西，你可以把它当做是根据它原有的呈现自动添加了大小约束。在我们的图片中，它将会自动调整自己的大小。

不再需要硬编码的宽高约束，你能够依赖本质内容大小的特性让图片自己适应大小。

让我们试试，在文档导航器中，找到按钮宽和高约束然后点击删除：

然后，更新控制器里所有的视图的排版位置通过点击右下角的三角图标按钮，然后选中 All Views\Update Frames 。

编译并运行，体验下你的按钮吧！

给屏幕添加图片

有时候，你也许想给屏幕添加一些图片让它跟按钮有些不同。在这个情况下，你可以使用一个 UIImageView 对象。在对象库里找到 Image View 并且拖拽到屏幕中。

你将会使用这个 Image View 对象来创建边框，并且给它设置尺寸和摆放在屏幕的顶端，拉伸至屏幕的两端。对底部的 Image View 对象采取同样的操作。

移动您的标签来腾出点空间。最简单方式是更新它的约束通过在尺寸检查点击编辑按钮。选中 Time 表情，然后在约束一栏找到 Size Inspector(尺寸检查器)，点击 Top Space to 编辑按钮设置 Top Layout Guide 约束。

现在，为顶部的 Image View 设置约束。点击右下角的方框按钮。首先，确保取消勾选了 Constrain to margins 这一项。接着，设置 左, 上, 和 右横杆 的值都为 0。最后，检查 高 约束。确保值为 22。一旦完成，点击 Add 4 Constraints。

如果你看到橘色参考线，请参考上面的 update the frames 方法。

对底部的 Image view 做相同的操作。尝试着参照顶部的 Image View 约束设置。如果你觉得有困难，参考以下剧透。

解决方案：

点击方块按钮。首先，确保取消勾选了 Constrain to margins 这一项。接着，设置 左, 下, 和 右横杆 的值都为 0。最后，检查 高 约束。确保值为 22。一旦完成，点击 Add 4 Constraints。

确保操作了 update the frames。

在检查器的顶部选中 Image View 选项，设置 Image 属性为 chekecker_top.png。设置 Mode 属性为 Aspect Fill。

对底部的 Image View 做相同操作，除了 Image 设置为 checker_bottom.png。

通过编程设置背景色

你不需要总是在故事板修改改变您的应用外观。让我通过代码来修改背景色成紫色。尝试在 ViewController.swift 里的 viewDidLoad() 函数内加入以下代码：

view.backgroundColor = UIColor.purpleColor()

上面这行代表是设置视图的属性 backgroundColor，将 UIColor 返回的 purpleColor() 赋值给背景色。

运行应用并且看下界面是怎样的。

尽管这验证了概念，但是它仍然看起来不是很美观。然而，幸运的是，你可以设置视图的背景为一张图片。让我再一次通过编程实现。将下面的代码替换掉设置紫色背景的代码：

view.backgroundColor = UIColor(patternImage: UIImage(named: "bg_tile.png")!)

这段代码表示设置一张平铺的图片当做视图背景。运行应用看看是什么样的。

继续为你的标签通过编程设置它们的背景。要实现这一点，编写以下两个代码：

scoreLabel.backgroundColor = UIColor(patternImage: UIImage(named: "field_score.png")!)
timerLabel.backgroundColor = UIColor(patternImage: UIImage(named: "field_time.png")!)

运行应用看看是什么样的。

添加声音

音乐和音响效果都是很好的特性。这就是该应用最后一件缺失的东西！

首先，你需要一些音频文件。下载这些音频文件，解压文件，并且将3个音频文件拖放到您的项目中。

这三个音频文件分别是：背景音乐，按钮点击的蜂鸣声，并且是标签倒计时每一秒的声音，这是为了让用户体验更好！

通过控制器的代码来控制背景音乐，所以开打 ViewController.swift。在顶部加入下面这句代码：

import UIKit

你还需要导入一个重要的 AVFoundation 框架，该框架是苹果提供用来负责音频播放的。在刚刚导入的 UIKit 下面添加代码：

import AVFoundation

就像导入 UIKit 框架能够让你使用 UIButton 和 UILabel 一样，导入 AVFoundation 能够让你使用非常强大实用的 AVAudioPlayer 类。接着，你将会需要为3个音频文件设置3个实例变量。

var buttonBeep : AVAudioPlayer?
var secondBeep : AVAudioPlayer?
var backgroundMusic : AVAudioPlayer?

因为 AVAudioPlayer 音频播放器有可能是不可用的，因此变量需要定义成可选类型。

接着，你需要在 viewDidLoad 函数上面添加下面这个辅助函数。

  func setupAudioPlayerWithFile(file:NSString, type:NSString) -> AVAudioPlayer?  {
    //1
    let path = NSBundle.mainBundle().pathForResource(file as String, ofType: type as String)
    let url = NSURL.fileURLWithPath(path!)
 
    //2
    var audioPlayer:AVAudioPlayer?
 
    // 3
    do {
      try audioPlayer = AVAudioPlayer(contentsOfURL: url)
    } catch {
      print("Player not available")
    }
 
    return audioPlayer
  }

该方法将会返回一个 AVAudioPlayer 对象 (通过 -> 符号定义返回类型)，并且接收两个参数：一个文件名和文件类型。让我们看看这段代码干了啥。

  1. 你需要知道音频文件的完整路径，NSBundle.mainBundle() 会告诉你在项目的哪个地方。AVAudioPlayer 需要知道资源的路径通过设置 URL，所以完整的路径需要转换成 URL 格式。

  2. audioPlayer 是个可选类型。因为 AVAudioPlayer 在实例化的时候，依赖的设备可能有不支持该功能现象。

  3. 这是你即将创建 AVAudioPlayer 的地方。创建这个对象的时候也许会抛出异常，你定义了一个 block 通过 do 关键字。接着你尝试创建 player对象。如果 player 是不能够被创建的时候，你将会接收一个错误。此时，在一个真正的应用中打印错误的信息，将会在 block 处理异常。这种错误处理方式在 Swift 2.0 里出现。

  4. 如果一切顺利，将会返回一个 AVAudioPlayer 对象!

现在你有一个方便的函数来设置 AVAudioPlayer 对象，是时候使用它了！添加以下代码到 viewDidLoad() 函数里的 setupGame() 之上:

if let buttonBeep = self.setupAudioPlayerWithFile("ButtonTap", type:"wav") {
  self.buttonBeep = buttonBeep
}
if let secondBeep = self.setupAudioPlayerWithFile("SecondBeep", type:"wav") {
  self.secondBeep = secondBeep
}
if let backgroundMusic = self.setupAudioPlayerWithFile("HallOfTheMountainKing", type:"mp3") {
  self.backgroundMusic = backgroundMusic
}

现在你创建了几个 players。如果 players 能够创建成功，对象将会赋值给实例变量。

此刻在 viewDidLoad，你将会有3个音频对象能够被你的代码调用！

第一个音频对象，buttonBeep，当按钮被点击时播放。更新 buttonPressed 函数来播放声音：

buttonBeep?.play()

通过在变量名后标记一个问号，表示你尝试通过可选的方式调用一个函数。如果它是一个对象，函数将会被调用。否则，代码将会被忽略。

那里还有2个其他的音频对象。 secondBeep 将会在每一次倒计时后播放。在 subtractTime() 函数里调用播放声音的代码。在 if 语句之前调用。

secondBeep?.play()

已经接近完成！

最后一步是添加背景音乐。在每次新的游戏开始后播放，添加播放代码到 setupGame() 函数的最下面：

backgroundMusic?.volume = 0.3
backgroundMusic?.play()

你可以调整背景音乐的音量使蜂鸣声能够被听到。修改 backgroundMusic 的 volume 属性能够做到这一点。它的值能够被设置成0(静音)至1.0(最大声)，但是0.3应该比较好。

最后一次运行游戏并且体验您的劳动成果！
